# Content Checklist

Use this before promoting a draft into `_posts/`.

- strong title
- concise description
- clear thesis and argument flow
- cover image URL present
- `image_alt` present
- `fact_check_status` set to `complete` or `not-needed`
- taxonomy normalized
- technical claims source-checked where needed
- internal structure reads like a native site article
- post passes `make validate-draft`
- post passes `make qa-publish`
- repo passes `make check`
